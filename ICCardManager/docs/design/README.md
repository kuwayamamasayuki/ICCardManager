# 設計書

本フォルダには、交通系ICカード管理システム：ピッすいの設計書一式を格納しています。

## 設計書の歩き方

### はじめて読む人へ

以下の順序で読むと全体像を効率よく掴めます。

1. **[01_システム概要設計書](01_システム概要設計書.md)** — 目的・背景・技術スタック・制約
2. **[04_機能設計書](04_機能設計書.md) §1 機能マップ** — どんな機能があるか俯瞰
3. **[02_DB設計書](02_DB設計書.md) §1 はじめに** — データの全体像と3大前提
4. **[03_画面設計書](03_画面設計書.md) §1 画面遷移図** — UI の全体像
5. 興味のある機能の詳細章へ

### 目的別ガイド

| やりたいこと | まず読むべき |
|---|---|
| 新機能を追加したい | `04_機能設計書.md` の該当機能 → `05_クラス設計書.md` の関連クラス |
| DBスキーマを変更したい | `02_DB設計書.md` §3 テーブル定義 → §4 マイグレーション履歴 |
| 画面を改修したい | `03_画面設計書.md` の該当画面 → `05_クラス設計書.md` の該当 ViewModel |
| 業務ロジックを理解したい | `04_機能設計書.md` |
| 処理の時系列を追いたい | `06_シーケンス図.md` |
| テストを追加したい | `07_テスト設計書.md` → `tests/` 配下の実コード |
| 共有フォルダモードを理解したい | `02_DB設計書.md` §10〜§11 → `04_機能設計書.md` §18 |
| システム全体を俯瞰したい | `01_システム概要設計書.md` |

## 共通の読み方

各設計書は以下の共通テンプレに従って構成されています:

- **§0 このドキュメントについて** — 対象読者/前提知識/答えられる問い/関連ドキュメント
- **§1 はじめに** — 全体像、機能マップ、読み方ヒント
- **§2 以降** — 本編(各設計書で異なる)
- **付録A** — 用語クイックリファレンス、もしくはサービス対応表等
- **付録B(一部のみ)** — 関連ドキュメント
