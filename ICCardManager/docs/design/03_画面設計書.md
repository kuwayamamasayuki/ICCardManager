## 3. 画面詳細設計

### 3.1 メイン画面（MainWindow）

#### 3.1.1 画面レイアウト

```
+----------------------------------------------------------------------------+
| [ヘッダー]                                                                  |
| 交通系ICカード管理システム：ピッすい    2024/04/01 09:30  [設定][帳票][管理] |
+----------------------------------------------------------------------------+
|                                                     |  [サイドバー]         |
|                                                     |  貸出中のカード       |
|                    [メインエリア]                    |  ・はやかけん H001    |
|                                                     |    4/1 9:00～         |
|                       💳                            |  ・nimoca N002        |
|             職員証をタッチしてください                |    4/1 8:30～         |
|                                                     |                       |
|                    残り 58 秒                        |  [警告]               |
|                                                     |  ・H003 残高不足      |
+----------------------------------------------------------------------------+
| [操作ヒント] 職員証→ICカードで貸出/返却   | [Esc]キャンセル               |
+----------------------------------------------------------------------------+
| 状態: 職員証タッチ待ち | 🔗 共有モード | 貸出中: 2枚                         |
+----------------------------------------------------------------------------+
```

> **共有モードインジケーター**: 共有フォルダモード（`database_config.txt` にパスが設定されている場合）で動作中の場合、ステータスバーに「🔗 共有モード」を表示する。スタンドアロンモード時は非表示。

> **ネットワーク切断警告バナー**: 共有フォルダモードでDB接続が失われた場合、メインエリア上部に警告バナー（`WarningType.DatabaseConnectionLost`）を表示する。背景色: #FFEBEE（薄い赤）。接続が回復すると自動的に非表示となる。

#### 3.1.2 領域説明

| 領域 | 説明 |
|------|------|
| ヘッダー | システム名、現在日時、各種ボタン |
| メインエリア | 状態アイコン、ステータスメッセージ、タイムアウト表示 |
| サイドバー | 貸出中カード一覧、警告メッセージ |
| カードダッシュボード | 登録カードの残高・貸出状態・最終利用日を一覧表示。並び替え機能あり |
| 操作ヒント | 操作方法のガイダンス |
| ステータスバー | 現在の状態、貸出中カード数、共有モードインジケーター |

#### 3.1.3 状態別表示

| 状態 | アイコン | メッセージ | 背景色 |
|------|----------|------------|--------|
| 職員証タッチ待ち | 💳 | 職員証をタッチしてください | 白 |
| 交通系ICカードタッチ待ち | 🚃 | 交通系ICカードをタッチしてください | 白 |
| 貸出完了 | 🚃→ | 貸出しました | #FFF3E0（薄いオレンジ、`LendingBackgroundBrush`） |
| 返却完了 | 🏠← | 返却しました | #E3F2FD（薄い水色、`ReturnBackgroundBrush`） |
| エラー | ⚠️ | エラーメッセージ | #FFEBEE（薄い赤、`ErrorBackgroundBrush`） |

> **色値の Single Source of Truth**: 上記の色値は `Resources/Styles/AccessibilityStyles.xaml` 内のブラシキーで定義されており、View からは `DynamicResource` で参照する。色値そのものを XAML/コードに直接書かないこと（Issue #1392）。

#### 3.1.4 カードダッシュボードの並び替え

カードダッシュボードは以下の基準で並び替え可能。

| 並び替え | 説明 |
|----------|------|
| カード名順 | カード種別・管理番号順（デフォルト） |
| 残高（少ない順） | 残高が少ないカードを先頭に表示 |
| 残高（多い順） | 残高が多いカードを先頭に表示 |
| 最終利用日順 | 最後に使用されたカードを先頭に表示 |

### 3.2 状態遷移図

```mermaid
stateDiagram-v2
    [*] --> WaitingForStaffCard: 起動

    WaitingForStaffCard --> WaitingForIcCard: 職員証タッチ
    WaitingForStaffCard --> 履歴表示: ICカードのみタッチ

    WaitingForIcCard --> Processing: ICカードタッチ
    WaitingForIcCard --> WaitingForStaffCard: タイムアウト(60秒)
    WaitingForIcCard --> WaitingForStaffCard: Escキー

    Processing --> LendComplete: 貸出処理成功
    Processing --> ReturnComplete: 返却処理成功
    Processing --> ErrorState: エラー

    LendComplete --> WaitingForStaffCard: 3秒後
    ReturnComplete --> BusStopInput: バス利用あり
    ReturnComplete --> WaitingForStaffCard: バス利用なし

    BusStopInput --> WaitingForStaffCard: 入力完了

    ErrorState --> WaitingForStaffCard: 3秒後
```

#### 状態説明

| 状態 | 説明 | タイムアウト |
|------|------|--------------|
| WaitingForStaffCard | 職員証タッチ待ち | なし |
| WaitingForIcCard | ICカードタッチ待ち | 60秒 |
| Processing | 処理中 | なし |
| LendComplete | 貸出完了 | 3秒 |
| ReturnComplete | 返却完了 | 3秒 |
| ErrorState | エラー表示 | 3秒 |
| BusStopInput | バス停入力待ち | なし |

#### 設定項目

設定項目は `Models/AppSettings.cs` のプロパティ順に対応する。

| 項目 | プロパティ | 設定値 | デフォルト |
|------|-----------|--------|------------|
| 残高警告閾値 | `WarningBalance` | 0～20,000円 | 10,000円 |
| バックアップ先フォルダ | `BackupPath` | 任意のフォルダパス | アプリケーションフォルダ |
| 文字サイズ | `FontSize` | 小/中/大/特大 | 中 |
| 最終VACUUM実行日 | `LastVacuumDate` | 自動記録（UI非表示） | NULL |
| メインウィンドウ位置・サイズ | `MainWindowSettings` | 自動記録（UI非表示） | 既定 |
| 音声モード | `SoundMode` | 効果音/音声（男性）/音声（女性）/無し | 効果音 |
| トースト通知の表示位置 | `ToastPosition` | 右上/左上/右下/左下 | 右上 |
| 部署種別 | `DepartmentType` | 市長事務部局（役務費）/企業会計部局（旅費） | 市長事務部局 |
| 返却時のバス停入力スキップ | `SkipBusStopInputOnReturn` | 有効/無効 | 無効 |
| 帳票出力先フォルダ | `ReportOutputFolder` | 任意のフォルダパス | 空（毎回ダイアログで選択） |
| データベースパス | （`database_config.txt`） | フォルダパス（UNCパス対応） | 空（ローカルデフォルト） |

> `LastVacuumDate` および `MainWindowSettings` はアプリが内部的に保存する状態であり、設定ダイアログには表示しない。
> データベースパスは settings テーブルではなく `C:\ProgramData\ICCardManager\database_config.txt` で管理する。

> **データベースパス設定**: データベースファイルの保存先フォルダを指定する。UNCパス（例: `\\server\share\ICCardManager`）を指定すると共有フォルダモードとなり、複数PCからの同時利用が可能となる。空の場合はローカルの`%PROGRAMDATA%\ICCardManager`が使用される。フォルダ選択ダイアログで指定可能。設定変更後はアプリケーションの再起動が必要。

- 出力先フォルダの指定（共有フォルダ UNC パス対応）
- 帳票の一括作成

- 「出力フォルダを開く」ボタンで保存先をエクスプローラーで開く

#### Visibility 制御の注意点

「作成結果」GroupBox の表示は `ReportViewModel.HasCreatedFiles`（`bool` プロパティ）でバインドする。`CreatedFiles.Count` を直接 `BooleanToVisibilityConverter` に渡しても WPF 標準の Converter は `int` を `bool` に変換できず常に `Collapsed` となるため使用しない（Issue #1410）。`CreatedFiles.CollectionChanged` を購読して `HasCreatedFiles` の `PropertyChanged` を発火させる仕組み。

### 5.2 ダイアログの初期フォーカス（Issue #1277）

ダイアログ起動時の初期フォーカス位置を明示的に指定し、キーボード操作ユーザー・スクリーンリーダーユーザーが操作開始位置を見失わないようにする。

| ダイアログ | 初期フォーカス先 | 指定方法 |
|---|---|---|
| CardManageDialog | カード一覧 (`CardDataGrid`) | `FocusManager.FocusedElement` 属性 |

> ```xml
> <Window FocusManager.FocusedElement="{Binding ElementName=FirstInputControl}">
>   <Grid>

> </Window>

### 5.3 ダイアログ別アクセスキー一覧（Issue #1276）

WCAG 2.1 SC 1.3.1（情報および関係性）準拠のため、主要ダイアログではラベルに `WPF Label` コントロール + `Target` 属性を用いて、スクリーンリーダーがラベル＝コントロールの関連付けを認識できるようにしている。同時に Alt+キーでフォーカス移動できるアクセスキーを付与している。

> 設計原則:
> - アクセスキーは **ダイアログ内でユニーク**（同じ Alt+文字を複数に割り当てない）にする
> - 極力、項目名の最初のローマ字（日付=D、摘要=S 等）を使用して覚えやすくする
> - アクセスキーは WPF の `_` プレフィックス表記で指定（例: `Content="日付(_D):"`）
> - `Target="{Binding ElementName=...}"` でラベルと入力コントロールを結びつける
> - `AutomationProperties.Name` と併用して、スクリーンリーダーからは自然な読み上げ、視覚利用者には Alt キーで操作可能にする

### 5.4 実装上の注意

- WPF では `Label.Content` 内に `_X` を書くと X がアクセスキーになり、表示時は X に下線が引かれる。リテラルなアンダースコアを表示したい場合は `__`（二重アンダースコア）を使う
- `Target="{Binding ElementName=xxx}"` を設定しないと、アクセスキーで Focus は移るが AutomationPeer からの「ラベル → コントロール」参照が成立しない。**必ず Target を指定すること**
- 動的生成される `ItemsControl`/`ListView` 内の要素は `x:Name` による単一指定ができないため、`AutomationProperties.Name` による代替ラベル付けで補う
- アクセスキーが衝突した場合、Alt+キー押下で最初のコントロールにフォーカスが移り、再度 Alt+キーで次のコントロールへ循環する。UI の混乱を避けるため、**ダイアログ内で重複させない**

### 5.5 入力検証エラーの視覚表示・フォーカス移動（Issue #1279）

入力ミスに対するフィードバックを以下の3層で提供する:

**1. リアルタイム視覚フィードバック（WPF ValidationRule）**
- TextBox の `Binding` に `NumericRangeValidationRule` を組み込み、`Validation.HasError=true` 時に `AccessibilityStyles.xaml` の暗黙 Style が赤枠（2px）+ ⚠アイコンを表示
- 色覚多様性対応: 色だけでなく **枠線の太さ（1px→2px）** と **⚠アイコン** も変化させる
- ⚠アイコンは `AutomationProperties.Name="入力エラーあり"` 付きでスクリーンリーダーが「ここにエラーがある」と読み上げ可能
- ToolTip で具体的なエラーメッセージを表示

**2. 保存時のフォーカス自動移動**
- ViewModel の `Validate()` / `SaveAsync()` が最初のエラー発生プロパティ名を `FirstErrorField` に記録
- Dialog のコードビハインドは以下のタイミングで該当 TextBox に `Focus()` + `SelectAll()` を実行:
  - `ContentRendered`（初期表示で既にエラー状態の場合）
  - 保存ボタン押下で `CanSave=false` の場合（SettingsDialog のみ。LedgerRowEditDialog は Save ボタン自体が無効化されるため初期フォーカスのみ）
- 入力中（キーストロークごと）には焦点を奪わない — ユーザー体験を妨げないため明示的なイベントでのみ移動する

**3. 統一的なエラーメッセージ（Issue #1275 の3要素）**
- 「何が」「なぜ」「どうすれば」の3要素をメッセージに含める
- 例: 「残額警告しきい値が-100で下限を下回っています。0以上の値を入力してください。」

**実装ガイド:**
```xml
<!-- TextBox に ValidationRule を組み込む例 -->
<TextBox x:Name="WarningBalanceTextBox">
    <TextBox.Text>
        <Binding Path="WarningBalance" UpdateSourceTrigger="PropertyChanged">
            <Binding.ValidationRules>
                <validation:NumericRangeValidationRule
                    Min="0" Max="20000"
                    FieldName="残額警告しきい値"/>
            </Binding.ValidationRules>
        </Binding>
    </TextBox.Text>
</TextBox>
```

```csharp
// ViewModel 側で FirstErrorField を設定
private void Validate()
{
    FirstErrorField = null;
    if (Balance < 0)
    {
        ValidationMessage = "…";
        FirstErrorField = nameof(Balance);  // Dialog が BalanceTextBox.Focus() を実行
        return;
    }
    // …
}
```

```csharp
// Dialog で FirstErrorField に応じてフォーカス
private void FocusFirstErrorField()
{
    Control? target = _viewModel.FirstErrorField switch
    {
        nameof(LedgerRowEditViewModel.Summary) => SummaryTextBox,
```

### 5.6 ダイアログ最小サイズと低解像度対応（Issue #1280）

ノート PC など 1366×768 環境で OK/キャンセルボタンが画面外に隠れない設計が必要。

**必須設定:**
- **全ダイアログに `MinHeight` / `MinWidth` を設定する**
- `MinHeight ≤ 720`（タスクバーを除いた 1366×768 の実用高さ）
- `MinWidth ≤ 1300`

**入力フォーム型ダイアログの構造:**
1. Window ルートに `MinHeight` / `MinWidth` を設定
2. `ResizeMode="CanResize"` でユーザーが任意にリサイズ可能にする
3. 入力フィールド領域を `ScrollViewer` で囲み、縦方向スクロールを許可
4. ボタンエリアは `Grid` の最下行（`RowDefinition Height="Auto"`）に配置してフッター固定
5. 外側 `Grid` の入力フォーム行は `Height="*"` にし、ScrollViewer が可変領域を占有

**実装例（LedgerRowEditDialog）:**
```xml
<Window MinHeight="400" MinWidth="500" ResizeMode="CanResize" ...>
  <Grid Margin="20">
    <Grid.RowDefinitions>
      <RowDefinition Height="Auto"/>  <!-- Row 0: 見出し等 -->
      <RowDefinition Height="*"/>     <!-- Row 1: 入力フォーム（可変・スクロール） -->
      <RowDefinition Height="Auto"/>  <!-- Row 2+: 警告・ボタン（固定） -->
    </Grid.RowDefinitions>
    <ScrollViewer Grid.Row="1" VerticalScrollBarVisibility="Auto">
      <Grid><!-- 入力フィールド --></Grid>
    </ScrollViewer>
    <!-- ボタン行は下固定 -->
  </Grid>
</Window>
```

**`SizeToContent="Height"` の小ダイアログ:**
- 確認ダイアログ等は `SizeToContent="Height"` + 固定 Width が適切なケースがある
- この場合も `MinWidth` と（コンテンツ最小高さを下限とする）`MinHeight` は明示する

**Escape キーによるキャンセル:**
- キャンセルボタンに `IsCancel="True"` を設定することで WPF は自動的に Escape キーで Click イベントを発火する
- 全ダイアログのキャンセル/閉じるボタンにこれが設定されていることを確認

**推奨初期サイズ:**
- 初期サイズは通常の表示（1920×1080 等）で見やすい値にする
- 初期サイズが `MinHeight` を大きく上回っても問題ない（ユーザーがリサイズ可能）
- ただし初期 Height が 1366×768 の実用高さを超える場合は、適切な初期値に抑える（SettingsDialog は 820 → 680 に調整）

### 5.7 AutomationProperties 整備方針（Issue #1278）

WCAG 2.1 AA 達成基準 4.1.2（名前・役割・値）準拠のため、全コントロールに適切な `AutomationProperties.Name` / `HelpText` を付与する。

**原則:**
- **Name**: コントロールの簡潔な名前（スクリーンリーダーがフォーカス時に読み上げる短い識別）
- **HelpText**: Name だけでは分からない補足説明（操作方法・意味・注意点）
- **LiveSetting**: 動的変化するコンテンツ（ステータスメッセージ・件数表示）には必ず設定
  - `Assertive`: 緊急度高（エラー・警告）
  - `Polite`: 緊急度低（件数更新・完了通知）
  - `Off` (既定): 能動的な変化通知なし

**件数表示の例（Issue #1278）:**
`Common/CountToAccessibilityTextConverter` は件数を自然言語に変換する専用コンバーターで、0 件時は「{主語}はまだありません。新規登録してください」、1 件以上時は「{主語}は{N}件です」を返す。`StringFormat=' ({0}件)'` のような視覚向け表示とは別に、`AutomationProperties.Name` にこのコンバーターをバインドしてスクリーンリーダー専用の読み上げを行う。

**トグル/選択状態の例（ReportDialog 先月/今月）:**
Button が疑似的にトグル動作する場合、`DataTrigger` で選択状態に応じて `AutomationProperties.Name` / `HelpText` を切り替え、「先月（選択中）」のように状態を含める。RadioButton や ToggleButton を使える場合はそちらを優先するが、既存 UI との整合のため Button のまま拡張する場合はこの方式を採用する。

**空状態・エラー状態の文言:**
「(0件)」や単なる「エラー」のような曖昧表現を避け、次に取るべきアクションまで含める（例: 「登録カードはまだありません。新規登録してください」、「ネットワーク共有に到達できません。ネットワーク接続を確認してください」）。

**動的 TextBlock に Name を付けない原則（Issue #1468）:**

`TextBlockAutomationPeer.GetNameCore` は既定で `Text` を返すため、バインディングや code-behind で `Text` が動的に書き換わる `TextBlock` には **`AutomationProperties.Name` を付与しない**。Name を明示すると Name 値が UIA で優先されて `Text` の動的更新がスクリーンリーダーへ通知されなくなる。代わりに以下の組み合わせで補強する。

| パターン | Name | HelpText | LiveSetting | 例 |
|---------|------|----------|-------------|-----|
| 静的ラベル（`Text` 固定） | 任意（同じ値で重複しないように） | 必要に応じて補足 | 不要 | `<TextBlock Text="操作内容:" />` |
| 動的ステータス（成功・失敗） | **付けない** | 状態の意味を補足 | `Assertive` | `StaffAuthDialog.StatusText` |
| 動的カウントダウン・進捗 | **付けない** | 値の意味を補足 | `Polite` | `StaffAuthDialog.TimeoutText` / `OperationLogDialog.PageInfo` |
| 値だけが入る Run 構成 | **付けない** | 構造の説明 | `Polite`（変化があれば） | `OperationLogDialog` のページ番号表示 |

**カバレッジ回帰防止:**
- `OperationLogDialog` / `StaffAuthDialog` は `DialogAutomationPropertiesCoverageTests`（Issue #1468 / UT-058a）で `Name` / `HelpText` の最低出現回数とキー操作の Name、`LiveSetting` の付与を静的検査する。
- 別ダイアログでカバレッジが薄いと判明した場合は本テストの `MinimumNameCounts` / `MinimumHelpTextCounts` に新ダイアログを追加して固定する。

### 5.7.1 スクリーンリーダー対応（Issue #1468 / #1509）

#### LiveRegion 即時通知の仕組み

`StaffAuthDialog` の `StatusText` には `AutomationProperties.LiveSetting="Assertive"` を付与し、スクリーンリーダー（NVDA / Windows ナレーター）で認証ステータスの変化が即時通知されるよう設計している。WPF UI Automation の LiveRegion は「既に可視な要素の Text が変化したとき」に通知されるため、以下の 2 点を実装上の必須条件とする：

1. **`StatusBorder` は常時可視化**: 初期 `Visibility="Collapsed"` にすると AutomationTree から除外され、Collapsed → Visible 遷移は LiveRegionChanged を発火させない。`Background="Transparent" BorderThickness="0" MinHeight="44"` で視覚的に中立な状態で常時可視化し、`ShowStatus()` 呼び出し時に背景色・枠を切り替える。

2. **`UIElementAutomationPeer.RaiseAutomationEvent` の明示呼び出し**: `TextBlock.Text` 更新だけでは LiveRegionChanged の発火が保証されない（WPF のバージョン・コントロールツリーの状態に依存）。`ShowStatus()` の末尾で必ず以下を呼ぶ：

   ```csharp
   var peer = UIElementAutomationPeer.FromElement(StatusText)
              ?? UIElementAutomationPeer.CreatePeerForElement(StatusText);
   peer.RaiseAutomationEvent(AutomationEvents.LiveRegionChanged);
   ```

#### 認証成功時もステータス表示する設計

認証成功時は効果音（Notify）のみで即座にダイアログを閉じる従来仕様だったが、スクリーンリーダー利用者には音声フィードバックが不十分だった。Issue #1509 で「成功時にも `ShowStatus($"認証に成功しました（{職員名}）")` を呼び、700ms 表示してから閉じる」設計に変更。タイムアウト失敗側の「1 秒表示 → 閉じる」と同じ `CloseAfterDelay(TimeSpan, bool)` ヘルパで実装を共通化。
