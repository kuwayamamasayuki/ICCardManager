| 項目 | 内容 |

## 1. ドキュメント体系

本プロジェクトのドキュメントは、目的別に3種に分類されます。

```
docs/
├── design/            ← 開発ドキュメント(本ドキュメントシリーズ)
├── manual/            ← 利用者向けドキュメント
└── (プロジェクトルート) ← プロジェクト管理ドキュメント
```

| 分類 | 配置 | 対象読者 | 管理方針 |
|------|------|----------|----------|
| **開発ドキュメント** | `docs/design/` | 開発者・保守担当 | コードと同じPRで更新。本シリーズ(00〜08) |
| **利用者向けドキュメント** | `docs/manual/` | エンドユーザー・管理者 | 機能追加・UI変更時に同期更新 |
| **プロジェクト管理ドキュメント** | プロジェクトルート | 開発者・監査担当 | リリース時・配布方法変更時に更新 |

## 3. ドキュメント更新時のルール

### 3.1 開発ドキュメント(設計書)の更新

- **コードと同じPRで更新**する。コード変更だけを先行マージせず、設計書との整合性を維持する
- **章構成テンプレ**: 新規設計書を追加する場合は、既存設計書と同じ§0冒頭テンプレ(対象読者/前提知識/答えられる問い/関連ドキュメント)に従う
- **番号順**: 既存の連番(01〜08)を維持。新規追加時は既存番号を変更しない
- **相互参照**: 他の設計書との関連箇所は付録B等で明示する

### 3.2 利用者向けドキュメントの更新

- **UIや機能変更があれば即同期**する
- Markdown と Word(.docx)の両方を更新する(配布は .docx だが、レビューしやすさのため .md も維持)
- スクリーンショットの差し替えを忘れない

### 3.3 テスト設計書の更新

- テストケースの追加・修正を行ったら、対応するテストコードと同じPRで更新する
- `07_テスト設計書.md` は機能別に分割予定(Issue #1214)

## 4. マニュアル類の章構成指針

本章では、各マニュアルを改訂するときに目安とする **章構成の設計方針** を記します。現行のマニュアル本文は `docs/manual/` を参照してください。

### 4.1 ユーザーマニュアルの推奨章構成

```
ユーザーマニュアル
├── 1. はじめに
│   ├── 1.1 本マニュアルについて
│   ├── 1.2 システムの概要
│   └── 1.3 用語説明
├── 2. 基本操作
│   ├── 2.1 アプリケーションの起動
│   ├── 2.2 画面構成の説明
│   ├── 2.3 ICカードリーダーの使い方
│   └── 2.4 アプリケーションの終了
├── 3. 貸出・返却操作
│   ├── 3.1 交通系ICカードを借りる(貸出)
│   ├── 3.2 交通系ICカードを返す(返却)
│   ├── 3.3 バス停名の入力
│   └── 3.4 よくある状況と対処
├── 4. 履歴の確認
│   ├── 4.1 利用履歴を見る
│   ├── 4.2 履歴の修正
│   └── 4.3 履歴の削除
├── 5. 帳票作成
│   ├── 5.1 月次帳票の作成
│   └── 5.2 Excelファイルの確認
├── 6. 付録
│   ├── 6.1 キーボードショートカット
│   └── 6.2 エラーメッセージ一覧
└── 索引
```

### 4.2 管理者マニュアルの推奨章構成

```
管理者マニュアル
├── 1. はじめに (管理者の役割 / システム構成)
├── 2. 初期設定 (インストール / PaSoRi 設定 / 初期登録)
├── 3. 日常管理 (職員・カード追加 / バックアップ確認 / 操作ログ確認)
├── 4. 定期メンテナンス (DB最適化 / 古いデータ削除 / バックアップ管理)
├── 5. トラブル対応 (バックアップ復旧 / よくあるトラブル / ログ分析)
└── 6. 付録 (DB構造 / 設定ファイル一覧 / 連絡先)
```

### 4.3 用語の一貫性

マニュアル類では **「交通系ICカード」** と **「職員証」** を明確に区別して記述します。単に「ICカード」と書くと職員証と区別がつかないため避けます(`.claude/rules/development-conventions.md` に詳細)。

## 5. [参考] クイックスタートガイド設計(現在はユーザーマニュアルに統合)

> この章は過去の設計案のアーカイブです。現在の実装では `ユーザーマニュアル.md` 本編に統合されています。

A4両面1枚での構成案:

**表面(貸出/返却の手順)**
- 交通系ICカードを借りるとき: 職員証タッチ → 交通系ICカードタッチ → 「いってらっしゃい!」
- 交通系ICカードを返すとき: 職員証タッチ → 交通系ICカードタッチ → 「おかえりなさい!」→ バス停名入力(バス利用時)

**裏面(補足情報)**
- 履歴を確認するとき: 交通系ICカードだけをタッチ(職員証は不要)
- キーボードショートカット: F6=月次帳票, F7=カード管理, F8=職員管理, Esc=キャンセル
- 困ったときは: カードが読めない → 置き直す / 「未登録」と表示 → 管理者連絡 / その他 → 管理者連絡

## 7. ドキュメント管理

### 7.1 バージョン管理

本プロジェクトのドキュメントは Git で管理し、コードと同じリポジトリで版管理します。独立したバージョン番号は設けず、Git の履歴をバージョン履歴とします。

### 7.2 レビュー

- 開発ドキュメント: コード変更と同じPRでレビュー
- 利用者向けドキュメント: UX 変更や文言修正が含まれる場合、管理者または運用担当のレビューを推奨

### 7.3 配布状況

| ドキュメント | 配布先 | 状態 |
|-------------|--------|------|
| ユーザーマニュアル | 全職員 | ✅ 作成済み |
| ユーザーマニュアル概要版 | 全職員 | ✅ 作成済み |
| 管理者マニュアル | システム管理者 | ✅ 作成済み |
| 開発者ガイド | 開発担当者 | ✅ 作成済み |
| サードパーティライセンス一覧 | 開発担当者、監査担当者 | ✅ 作成済み |
| 配布手順書 | 開発担当者 | ✅ 作成済み |
| クイックスタートガイド | 全職員(カードリーダー設置場所に掲示) | ✅ ユーザーマニュアルに統合済み |
| トラブルシューティング | システム管理者、ヘルプデスク | ✅ ユーザーマニュアル / 管理者マニュアルに統合済み |
