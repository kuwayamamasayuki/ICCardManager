# 用語集

本システムおよび設計書で頻出する用語を集約したリファレンスです。各設計書の付録A「用語クイックリファレンス」の内容を統合し、全体横断で参照できるようにしています。

> **本書の位置づけ**: 各設計書を読むときに不明な用語があれば、まずここを引いてください。技術用語と業務用語の両方を含みます。

---

## 業務用語

| 用語 | 意味 |
|------|------|
| **交通系ICカード** | はやかけん、nimoca、SUGOCA 等、鉄道・バスの乗車に使う FeliCa カード。本システムの管理対象。**単に「ICカード」とは呼ばない**(職員証と区別できなくなるため) |
| **職員証** | 職員を識別するために使用する FeliCa カード。操作者の認証に使用 |
| **IDm** | FeliCaカードの製造ID。16進数16文字。本システムでは職員証・交通系ICカードの識別キーとして使用 |
| **貸出** | 職員に交通系ICカードを物理的に渡す操作。`ic_card.is_lent = 1` で表現 |
| **返却** | 職員から交通系ICカードを受け取る操作。利用履歴を読み取り `ledger` に記録する |
| **30秒ルール** | 誤操作救済のため、30秒以内に同一カードを再タッチすると前回と逆の処理を実行する機能。詳細: 04 §3 |
| **持ち替え** | 交通系ICカード待ち状態中に別の職員証がタッチされたとき、操作者情報を上書きする挙動。Issue #1211。詳細: 04 §2.3 |
| **物品出納簿** | 行政の金券類・物品の受払を記録する帳簿。本システムはこれをデジタル化したもの |
| **乗り継ぎ統合** | 連続する鉄道利用を1つの摘要にまとめる機能。Issue #484。詳細: 04 §5.4 |
| **残高不足パターン** | 運賃に対して残高不足のとき、不足分を現金でチャージして乗車する特殊処理。詳細: 04 §5.6 |
| **暗黙的ポイント還元** | `is_point_redemption` フラグが立たないまま金額マイナスで記録されるポイント還元。Issue #942。詳細: 04 §6 |
| **バス利用判別** | 乗車駅・降車駅が空欄かつチャージ/還元でない履歴をバス利用と判定するロジック。詳細: 04 §4 |
| **摘要** | 物品出納簿の「摘要」欄に出力される文字列。`ledger.summary` に保存。生成ルール: 04 §5 |

## データ・スキーマ用語

| 用語 | 意味 |
|------|------|
| **論理削除** | レコードを物理削除せず `is_deleted = 1` で無効化する方式。`staff` / `ic_card` で採用(過去履歴の表示時にデータが必要なため) |
| **物理削除** | 実際に DELETE 文でレコードを消す方式。`ledger` / `operation_log` は6年経過後に物理削除(監査期間経過後) |
| **マイグレーション** | スキーマ変更を連番クラスで管理し、起動時に `MigrationRunner` が未適用分を実行する仕組み。詳細: 02 §4 |
| **SequenceNumber** | `ledger_detail` の論理的な時系列番号。DBの物理カラムではなく SQLite の `rowid` から取得する仮想値 |
| **group_id** | 乗り継ぎ統合で使用する `ledger_detail` のグループ化キー。NULL 時は自動判定 |
| **undo_data** | 履歴統合操作の取り消し用 JSON。`ledger_merge_history.undo_data` に保存 |

## インフラ・運用用語

| 用語 | 意味 |
|------|------|
| **スタンドアロンモード** | 1台の PC 上でローカルの SQLite ファイルを使う標準運用形態 |
| **共有フォルダモード** | SMB 共有フォルダ上の SQLite ファイルを複数 PC(最大約20台)で共有する運用形態。`database_config.txt` で UNCパスを指定すると有効化。詳細: 02 §10〜§11、04 §18 |
| **busy_timeout** | SQLite のロック待機の最大時間。ローカル5000ms / 共有15000ms |
| **journal_mode** | SQLite のトランザクションジャーナル方式。本システムは DELETE を使用(WAL は SMB で不具合のため不可) |
| **PaSoRi** | Sony 製の FeliCa リーダー(RC-S300/RC-S380)。felicalib 経由でアクセス |
| **CardLockManager** | カードIDm 単位の排他制御を行うアプリ層のシングルトン。詳細: 04 §15 |
| **ヘルスチェック** | 共有モード時、30秒ごとに DB 接続の正常性を確認する機構 |

## 技術スタック用語

| 用語 | 意味 |
|------|------|
| **MVVM** | Model-View-ViewModel パターン。本システムの UI アーキテクチャ |
| **CommunityToolkit.Mvvm** | `[ObservableProperty]`, `[RelayCommand]` 等を提供する MVVM フレームワーク |
| **ClosedXML** | Excel ファイルの読み書きライブラリ。月次帳票出力に使用 |
| **felicalib** | FeliCa カード読み取り用のライブラリ |
| **xUnit / FluentAssertions / Moq** | 本プロジェクトの単体テストで使用するライブラリ |
| **SQLite Backup API** | DB 接続中でも安全にバックアップできる SQLite の API。本システムのバックアップ機能で使用 |

## 状態・フラグ用語

| 用語 | 意味 |
|------|------|
| **WaitingForStaffCard** | メイン画面の初期状態。職員証タッチ待ち |
| **WaitingForIcCard** | 職員証タッチ後の状態。交通系ICカードタッチ待ち。60秒タイムアウトあり |
| **is_lent** | `ic_card` の貸出状態フラグ。0=返却済、1=貸出中 |
| **is_lent_record** | `ledger` の「貸出中レコード」フラグ。貸出時に作成し返却時に解除 |
| **is_refunded** | `ic_card` の払戻済フラグ。Issue #530 |
| **is_bus** | `ledger_detail` のバス利用フラグ。バス判別ロジックの結果 |
| **is_point_redemption** | `ledger_detail` のポイント還元フラグ |
| **is_charge** | `ledger_detail` のチャージフラグ |
| **is_undone** | `ledger_merge_history` の取消済フラグ |
