# Issue #1407: 管理者マニュアル §9.1 「日次（自動）バックアップ」表記の実装乖離修正

## 背景

管理者マニュアル §9.1「定期メンテナンス」の表で、自動バックアップの頻度を **「日次」** と記載しているが、実装は **アプリケーション起動時** に作成する仕様。

該当箇所: `ICCardManager/docs/manual/管理者マニュアル.md` 行 1300〜1304

```markdown
| 頻度 | 作業内容 |
|------|----------|
| 日次 | （自動）バックアップ |
| 月次 | 帳票出力、残額確認 |
| 年次 | 古いデータの確認、ディスク容量確認 |
```

一方、同マニュアル内の他箇所（§3.3、§6.1、§6.3 のバージョンアップ補足）およびユーザーマニュアル §6.1 では正しく「起動時に自動でバックアップを作成」と記載されており、§9.1 のみが乖離している。

## 影響

PC を起動しっぱなしで運用する組織と、毎日電源を切る組織でバックアップ頻度が変わる。記載「日次」を真に受けた管理者は、長期間アプリを起動しない運用（例: 利用が稀な部門）で実際にはバックアップが作成されていないにも関わらず、日次で取得されていると誤解する恐れがある。

## 原因

§9.1 の表は管理者向けの「メンテナンス頻度の早見表」として「日次／月次／年次」の頻度区分で構成されているため、自動バックアップを「日次の定例作業」と分類した結果、実装挙動（イベント駆動：起動時）と乖離した。

## 修正方針

1. 表内の「日次」を **「起動時」** に置換
2. 表直下に **補足ブロック** を追加し、以下を明記
   - 自動バックアップは起動時に作成される
   - §3.3 / §6.1 への参照リンク（既存記述との整合）
   - 常時起動運用なら実質日次となるが、毎日アプリを起動・終了する運用ではその頻度に依存する点
   - 長期間アプリを起動しない場合は手動バックアップ（F6 システム管理画面）を推奨

頻度ヘッダ「頻度」自体は変更しない（依然として「いつ実施するか」の分類列として機能する）。

## 採用理由

Issue で示された 2 案のうち：

- **「起動時」と書く + 起動頻度依存の補足を添える**: 採用
- **「起動時（アプリ起動毎）」と一言で書く**: 表セル内に収めるには適しているが、頻度依存の運用上の含意（手動運用ではバックアップが取れない期間が発生し得る）が表だけでは伝わらない

採用案では、表セルは簡潔に「起動時」と表記し、運用上の重要な含意は表直下の補足ブロックで丁寧に説明することで、流し読み層と精読層の双方に対応する。
