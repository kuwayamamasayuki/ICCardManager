# Issue #1571: UNC パス推奨記載の削除（マップドドライブとの並列扱い化）

- **Issue**: [#1571 管理者マニュアルの記載内容の修正（UNCパス vs マップドドライブ）](https://github.com/kuwayamamasayuki/ICCardManager/issues/1571)
- **作成日**: 2026-05-22
- **種別**: ドキュメント修正のみ（コード変更なし）

## 背景

管理者マニュアル §2.3「2 台目以降の PC への導入」の「データベースの保存先」入力例で、UNC パスとマップドドライブの 2 形式を提示する際、UNC パスを「**推奨**」と表記し、補足では「マップドドライブはドライブレターが PC ごとに異なると運用が複雑になり、無人起動との相性が悪い場合があります」と書かれている。

しかし、想定環境（自治体・企業）ではシステム管理者が各 PC に対してドライブレターを統一的に割り当てており、PC ごとにドライブレターが異なる事態は発生しない。むしろ実運用ではマップドドライブの方が使われていることが多く、現状の「UNC パス推奨」記載は実態と乖離している。

コード本体は Issue #1559 で既に UNC パス／マップドネットワークドライブの両形式を等価にサポート済み（`DbContext.IsUncPath` と `IsMappedNetworkDrive` で同じ共有モードを発火）。よって本 Issue は純粋にドキュメントの整合性を整える修正となる。

## ゴール

1. 管理者マニュアル・ユーザーマニュアル内で UNC パスを「推奨」とする表現を削除する
2. UNC パスとマップドドライブを**並列**の選択肢として中立に提示する
3. 接続確認手順や用語集など、UNC パスのみを前提にしている記述にマップドドライブの記述を追加する
4. コードや単体テストは変更しない（既に両形式を等価サポート済みのため）

## 非ゴール

- コードロジックの変更（`DbContext` / `PathValidator` 等の判定ロジックは変更しない）
- 設計書（`docs/design/01-07`）の変更（実装仕様の記述であり「推奨」表現は使っていない）
- 開発者ガイドの変更（技術仕様の記述でありユーザー誘導文ではない）

## 修正内容詳細

### 1. 管理者マニュアル L273-276（表）

```diff
   | 形式 | 入力例 | 備考 |
   |------|--------|------|
-  | UNCパス（**推奨**） | `¥¥Server¥ICCardShare¥ICCardManager` | 全PCで同じ表記で統一できる |
-  | マップドドライブ | `K:¥ICCardShare¥ICCardManager` | 各PCで同じドライブレターへのマップが必要 |
+  | UNCパス | `¥¥Server¥ICCardShare¥ICCardManager` | 全 PC で同じ表記が使える |
+  | マップドドライブ | `K:¥ICCardShare¥ICCardManager` | システム管理者がドライブレターを統一している環境で利用可能 |
```

### 2. 管理者マニュアル L278（補足）

```diff
-  > **補足**: UNCパスを推奨します。マップドドライブは PC ごとにドライブレターが異なると運用が複雑になり、またログインユーザーによって接続状態が変わるため、無人起動（常駐起動）との相性が悪い場合があります。
+  > **補足**: 組織のシステム管理者によってドライブレターが全 PC で統一されている場合はマップドドライブを、ファイル共有を直接参照したい場合は UNC パスを指定してください。どちらの形式でも共有モードが自動的に有効化されます（→ [§2.5 共有モードについて](#25-共有モードについて)）。
```

### 3. 管理者マニュアル L1488（接続確認手順）

```diff
-4. **UNCパスでの接続確認**: エクスプローラーで `\\サーバ名\ICCardManager` にアクセスし、ファイルの作成・編集・削除が可能であることを確認
+4. **共有フォルダへの接続確認**: エクスプローラーで共有フォルダ（UNC パス例: `\\サーバ名\ICCardManager`、マップドドライブ例: `K:\ICCardManager`）にアクセスし、ファイルの作成・編集・削除が可能であることを確認
```

```diff
-| 共有モード | データベースを UNC パス（`\\サーバ名\共有名\iccard.db`）で指定したときに自動的に有効化される複数 PC 共有運用モード。最大約 20 台で同時利用できます（→ [§2.5 共有モードについて](#25-共有モードについて)） |
+| 共有モード | データベースを UNC パス（`\\サーバ名\共有名\iccard.db`）またはマップドネットワークドライブ（`K:\共有名\iccard.db` 等）で指定したときに自動的に有効化される複数 PC 共有運用モード。最大約 20 台で同時利用できます（→ [§2.5 共有モードについて](#25-共有モードについて)） |
```
