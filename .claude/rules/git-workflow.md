# Gitワークフロー（厳守）

## ブランチルール
- **mainブランチへの直接コミットは禁止**（バージョン更新等の軽微な変更も含む）
- すべての変更は必ずブランチを作成し、PRを経由してマージすること
- ブランチ命名規則: `fix/issue-XXX-短い説明`、`feat/issue-XXX-短い説明`、`docs/issue-XXX-短い説明`、`chore/短い説明`

## コミット時のステージング
- **`git add -A` や `git add .` は使用禁止** — 意図しないファイル（.env、credentials、大きなバイナリ等）が含まれるリスクがあるため
- 必ず `git add <ファイル名>` で個別にステージングすること
- 多数のファイルがある場合でも、変更内容を確認した上で個別に指定する
