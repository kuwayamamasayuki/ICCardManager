# Issue #1491 設計書: 開発者ガイド付録 C トラブルシューティングを現役パターンに更新

## 背景

`ICCardManager/docs/manual/開発者ガイド.md` 付録 C トラブルシューティング表に
以下の記載がある（line 1419）:

| 症状 | 原因 | 対処 |
|------|------|------|
| テスト失敗 | DBマイグレーション | テスト前に `ClearOperationLogs()` を呼ぶ |

調査の結果、`ClearOperationLogs()` というメソッドは **本体・テスト全体で 1 件も実在しない**
（grep 結果: 開発者ガイドの 1419 行目のみ）。

## 問題

1. **実在しないメソッドを参照している** — レガシーコード由来の参照と推定。新規開発者が
   この対処を試そうとして混乱する。
2. **現代のテスト隔離は完全に別パターン** — 各テストクラスが `:memory:` SQLite 接続を
   コンストラクタで作成し `IDisposable.Dispose` で破棄する xUnit パターン
   （`MigrationIdempotencyTests` 等が典型例）。手動 `ClearOperationLogs` のような
   グローバル状態クリアは不要かつ存在しない。
3. **マイグレーション関連の典型対処は別物** — Issue #1285 で導入された
   `MigrationIdempotencyTests` パターンが現代の調査起点。

## 採用方針

「行削除」ではなく「**現代のパターンに置き換え**」で更新する。理由:

- 今後同種のテスト失敗（マイグレーション周り）が発生したとき開発者が辿れる情報を
  残す価値が高い。
- 単に削除すると同じ症状で同じ古い記憶を辿って混乱するリスクが残る。

## 変更内容

### ファイル: `ICCardManager/docs/manual/開発者ガイド.md`

付録 C トラブルシューティング表を以下のように更新:

**現状（1 行）:**
> | テスト失敗 | DBマイグレーション | テスト前に `ClearOperationLogs()` を呼ぶ |

**更新後（複数行 + 補足）:**
- 「テスト失敗（マイグレーション関連）」: `MigrationIdempotencyTests` を参照点として
  `:memory:` SQLite + xUnit `IDisposable` の隔離パターンを案内
- 「テスト失敗（UI スレッド関連）」: `DbContextUiThreadGuardTests` を参照点として、
  Service 層の UI スレッド呼び出しを検出する仕組みを案内
- 既存の他 3 行（ビルドエラー / デバッグ時カード認識 / 本番カード認識）は維持

加えて、テーブル下に「現代のテスト隔離パターン」の簡潔な解説段落（5〜10 行）を
追加し、`ClearOperationLogs` 等の手動グローバルクリアは現代のテスト基盤では
**不要**であることを明示する。

### その他のドキュメント

`CHANGELOG.md` の Unreleased 「テスト強化」または「ドキュメント」セクションに
Issue #1491 のエントリを追加。

## 影響範囲

- **プログラム変更なし** → 単体テスト追加不要。
- **手動テスト不要** → ドキュメントの校正のみ。
- 既存のテストやコードへの参照は一切ない。

## ロールアウト

- 通常マージ。リリースノート不要（テスト強化または運用ドキュメント節）。
