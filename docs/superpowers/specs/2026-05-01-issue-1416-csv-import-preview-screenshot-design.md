# Issue #1416: 管理者マニュアル §5.6.5 / §6.4 CSVインポートのプレビュー画面のスクリーンショット追加

## 背景

管理者マニュアル §5.6.5「月途中からの履歴入力（CSVインポート）」の「CSVインポートの実行」サブセクション (lines 794〜824) と §6.4「データインポート」の「インポートの手順」 (lines 962〜974) は、CSV インポートのプレビュー機能を **「追加（緑）/修正（オレンジ）/スキップ（灰）/復元（青）」の色分けアクション付きの強力な機能** として説明している。にもかかわらずプレビュー画面そのもののスクリーンショットが存在せず、機能の便利さが伝わりにくい。

特に §5.6.5 は本アプリの初期導入時に紙の物品出納簿から過去履歴を移行する重要工程で、視覚的な手がかりがあると失敗のリカバリ判断もしやすくなる。

該当箇所: `ICCardManager/docs/manual/管理者マニュアル.md` §5.6.5 / §6.4

## 実装側の状態（事前調査）

### CSV インポートのプレビュー UI

`Views/Dialogs/DataExportImportDialog.xaml` の `Grid.Row="4"` に **インラインで展開** される (別ダイアログを開く構造ではない)。具体的には:

| 行 | 構造 |
|---|---|
| 39 | `<RowDefinition Height="2*" MinHeight="0"/>  <!-- Row 4: プレビュー結果＋エラー（リサイズ可能） -->` |
| 365-371 | 「プレビュー(_P)」ボタン (`PreviewImportCommand` バインド) |
| 376-380 | 「インポート実行」ボタン (`HasPreview` で活性化制御) |
| 423-489 | プレビュー結果セクション (サマリー + DataGrid) |

つまり 1 枚のスクリーンショットで「データ種別選択 + プレビュー結果 DataGrid + アクションボタン群」が同時に映る。Issue 概要の要求「プレビューダイアログの全体スクリーンショット」と「『インポート実行』『キャンセル』ボタンの位置関係がわかる構図」を 1 ファイルで満たせる。

## スコープ判断

§6.4 の説明文 (line 962-974) は §5.6.5 への参照リンクで詳細を委譲している (line 974: 「[5.6.5 月途中からの履歴入力（CSVインポート）](#565-月途中からの履歴入力csvインポート)」を参照してください)。プレビュー UI 自体は両節で同一を指すため、**1 枚の画像 (`import_preview.png`) を 2 箇所から参照** する形を採用する。

別構図 (例: §6.4 用にサマリーのみ抜粋した小さな画像) の追加撮影は YAGNI であり、撮影コストを上積みする割に視認性は変わらない。

## テスト方針

本 PR の変更は **Markdown 文書のみ** でコードロジック変更がないため、単体テスト対象外。

ユーザーへの動作確認依頼:

1. 当ブランチを checkout 後、テスト用 CSV を準備:
   - 4 アクション全種 (追加 / 修正 / スキップ / 復元) が出現する状態が望ましい
   - 既存データと一部重複 (修正・スキップ用)、論理削除済みデータと重複 (復元用)、新規データ (追加用) を含む内容
   - 4 種すべてを 1 ファイルに揃えるのが難しい場合は、最低でも「追加」「修正」「スキップ」の 3 種が映ることを目標 (色分けの判読性確認)

2. PowerShell で:
   ```powershell
   .\tools\TakeScreenshots.ps1 -Only import_preview
   ```

3. 「データ入出力」画面 (F4) → CSV ファイルを選択 → 「プレビュー」ボタン → プレビュー結果 DataGrid が色分け付きで表示された状態で Enter（→ `import_preview.png` が保存される）

4. `ICCardManager/docs/screenshots/import_preview.png` を Markdown プレビューで参照確認:
   - §5.6.5 step 3 直後に表示されるか
   - §6.4 step 4 直後に表示されるか

5. 撮影画像の追加コミットは別途実施 (Issue #1415 と同パターン)
